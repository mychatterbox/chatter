메인보드 교체 후 리얼텍 랜 드라이버 설치가 안되는 상황.  
장치 관리자에서 인식하지만 느낌표 떠 있고, 속성을 확인하면 장치 상태가 'Windows가 이 디바이스의 클래스 구성을 설치 중입니다.(코드 56)' 로 표시됨.

- 수동으로 드라이버 설치 시 오랜 시간 동안 설치 시도하고, 결국 설치 실패  
- 리얼텍 랜드라이버 설치 프로그램을 실행하면 계속 설치 시도 하지만 결국 실패하고 "The Realtek Network Controller was not found. If Deep Sleep Mode is enabled Please Plug the Cable" 창 표시

검색해보면 이런저런 방법으로 해결했다는 여러가지 경험담이 나옵니다만  
혹시 vmware 설치해서 사용한 적이 있다면 (지금은 제거 했더라도)  
레지스트리에서 아래 항목을 삭제하고 드라이버 설치 재시도 해봅니다.  

```
컴퓨터\HKEY_LOCAL_MACHINE\SOFTWARE\Classes\CLSID\{3d09c1ca-2bcc-40b7-b9bb-3f3ec143a87b}
```

위 경로를 단계씩 찾아가지 말고 <mark>복사</mark>해서 레지스트리 편집기의 <mark>상단 주소 표시줄에 붙여넣으면</mark> 해당 위치로 <mark>바로 이동</mark>합니다.
